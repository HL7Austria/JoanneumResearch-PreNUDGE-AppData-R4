# Some citizen-facing answerOptions/answers deliberately use a German (de-AT) display text for codes from terminologies that have no official German designation (e.g. SNOMED CT), so the UI can render a plain-language answer without a separate translation lookup at runtime. The IG Publisher's display-name check flags this as a mismatch whenever a SNOMED-/LOINC-aware terminology service is available, since only the English canonical display is registered. This suppresses the IG Publisher build-time warning only - it does not affect runtime validation on a FHIR server that itself checks display names against a terminology service. Current example: Whooley questionnaire, SNOMED CT 373066001/373067005 ('Yes'/'No' shown as 'Ja'/'Nein').

Wrong Display Name 'Ja' for http://snomed.info/sct#373066001. Valid display is one of 2 choices: 'Yes (qualifier value)' (en) or 'Yes' (en) (for the language(s) 'en-US')
Wrong Display Name 'Nein' for http://snomed.info/sct#373067005. Valid display is one of 2 choices: 'No (qualifier value)' (en) or 'No' (en) (for the language(s) 'en-US')
